## 📈 整体架构

### 数据流
```
用户 → 购买号码 (S-06)
  ↓
订单保存 (localStorage)
  ↓
活跃订单页显示 (S-07)
  ↓
轮询获取 SMS (每 3 秒)
  ↓
收到验证码
  ↓
保存到历史 (S-08)
  ↓
导出/查看 CSV
```

### 技术栈
- **前端**: HTML5, CSS3, Vanilla JavaScript
- **后端**: Go (Gin Framework)
- **存储**: localStorage (前端)
- **通信**: HTTP 轮询（可升级 WebSocket）
- **集成**: 5SIM API

## 🔧 已知优化点

### 性能优化
1. **轮询优化**: 当前每个订单 3 秒轮询，10 个订单 = 3.3 req/s
   - 建议: 实现 WebSocket（S-11）
2. **localStorage 优化**: 200 条记录约 50KB
   - 建议: 超过 500 条迁移到 IndexedDB
3. **渲染优化**: 历史表格大量数据时使用虚拟滚动
   - 建议: 引入 virtual-scroll 库
