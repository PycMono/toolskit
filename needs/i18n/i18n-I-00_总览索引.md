# 模块化 i18n 架构 — 总览索引

## 核心痛点

| 现状 | 问题 |
|------|------|
| 单一大文件（如 `zh.json`） | 团队协作冲突、文件过大、难以按模块维护 |
| 无命名空间 | key 命名混乱、重复、无法按页面懒加载 |
| 无热重载 | 修改翻译需重启服务 |
| 模板函数单一 | 不支持插值、复数、嵌套 |

## 解决方案

**命名空间分片（Namespace Sharding）+ 懒加载 + 中间件注入**

## 技术栈

- **语言**: Go 1.21+
- **框架**: Gin v1.9+
- **模板**: html/template（原生）
- **i18n 文件格式**: JSON（命名空间分片）
- **依赖**: 仅标准库 + `fsnotify`（热重载可选）
