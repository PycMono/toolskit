## 设计决策

### 1. 命名空间策略（Namespace）
- 每个业务模块对应一个 JSON 文件
- key 格式: `namespace.section.key`（最多3级，避免过深）
- 跨模块引用: 统一放 `common` 命名空间

### 2. 语言检测优先级
```
URL param (?lang=zh-CN)
    ↓ 未命中
Cookie (lang=zh-CN)
    ↓ 未命中
Accept-Language Header
    ↓ 未命中
Default Language (en-US)
```

### 3. Fallback 策略
```
zh-TW → zh-CN → en-US → return key
```

### 4. 加载策略
- **启动时全量加载**（推荐生产环境）
- 可选：fsnotify 监听文件变化自动热重载（开发环境）
